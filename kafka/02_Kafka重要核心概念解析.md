# Kafka重要核心概念解析

## 基本信息
```
Apache Kafka本质上是一一款分布式、基于发布/订阅机制的消息系统,
主要使用用Scala语言言开发而而成的。
这里里里使用用了了『主要』两字,说明Kafka的源码并非非全部都使用用Scala语言言开发,
有相当一一部分代码是使用用Java语言言开发的,特别是针对于客户端的部分。

Kafka的底层实际上是非非常复杂的,里里里面面涉及到大大量量的模式、算法、并发处理理逻辑;
当然,还涉到很多Java NIO(如Selector、Channel、Buffer等)、零拷⻉贝、异步通信等模块。
```

## Kafka重要概念
```
生产者(Producer)

    它的主要工作就是源源不不断地生产出消息,然后发送给消息队列。生产者可以向消息队
    列发送各种类型的消息,如狭义的字符串消息(正如第2节课的示例所演示的
    那样,发送了了一个字符串消息),也可以发送二进制消息。
    
消费者(Consumer)

代理(Broker)

    在消息队列列领域中,它指的其实就是消息队列列产品本身,比比如说在Kafka这个领
    域下,Broker其实指的就是一一台Kafka Server。换句句话说,我们可以将部署
    的一一台Kafka Server看作是一一个Broker,就是这样简单。那么从流程上来说,
    生生产者会将消息发送给Broker,然后消费者再从Broker中拉取消息。
```
